Useful Linux commands for developers

15 April 2014

System Information

Processor Info:

cat /proc/cpuinfo

Memory Info:

cat /proc/meminfo

Version of your kernel and the GCC compiler:

cat /proc/version

File and display commands

Lists files with permissions, shows hidden files, displays them in a column format, and suppresses group information.

ls -laxo

Clear screen:

clear

Remove directory with all content inside:

rm -rf example

User access

Change a bin access mode to be executable:

chmod +x file.bin

Switch to super user:

su

Run a command using super user:

sudo (command)

Exit from Super user mode or close terminal window:

exit

Open application using Super user:

sudo open -a "app_name"

Process management

List active processes:

ps -a

discover PID of a process:

pidof java

Kill process by PID:

kill 2581

Kill process and force shutdown:

kill -9 2581

Kill all processes with same name:

killall java

GIT

adds all current changes to staging area

git add -all

commits current changes from staging area to local branch

git commit -m "commit message"

pushes local commits to remote branch

git push

Pushes the commits from your local master branch to a (possibly new) remote branch my_test and sets up master to track origin/my_test.

git push -u origin master:my_test

node.js

Remove all globally installed packages with npm:

sudo npm -g ls | grep -v 'npm@' | awk '/@/ {print $2}' | awk -F@ '{print $1}' | xargs sudo npm -g rm

Purge NodeJS installation

sudo apt-get remove --purge nodejs

Solve Permission access issues when running without sudo:

sudo chown -R $USER /usr/{local/share/man,local/bin,lib/node}

and

sudo chown -R $USER
