Angular 2

Finally the most awaited Angular2 has been released under beta version. And as it was previously stated, it promised to carry rather steep changes compared to version 1, and from what we can see from Angular's version 2 documentation

From what it seems to be the Angular 2 advocates the use of a further layer of abstraction over plain javascript. Even though you still can code using plain JS with Angular 2, the standard way seems to be using a type system. Giving it a type system has many advantages for developers, such as: compile-time syntax corrections, type safety checking, and much more possibilities for complex refactorings using an IDE. While some example uses Typescript, Angular works equally well with ES5, ES6 and Dart as well.

Another reason for adopting angular 2 is performance. Having a performance way more times better than Angular 1 as based on these benchmarks is surely welcome. Many improvements were done to improve memory usage and processing time. The digest cycle was replaced by the use of a change detection algorithm that walks the DOM tree. Other optimizations allow Angular 2 to detect precisely when the model has changed.
